# Java

## Cuándo usarla

- Backend empresarial (Spring) donde el mercado laboral lo pide masivamente.
- Aprender POO "canónica": interfaces, herencia, patrones clásicos se enseñan en Java por algo.

## Cuándo NO usarla

- Scripts y prototipos rápidos: la ceremonia (clases, boilerplate) pesa demasiado.
- Frontend web.

## Ventajas

- JVM madura: rendimiento sólido, portabilidad real, tooling industrial (Maven/Gradle).
- Tipado estático estricto: gran parte de los errores se atrapan al compilar.
- Enorme demanda laboral en banca, gobierno y empresa grande.

## Desventajas

- Verboso frente a lenguajes modernos, aunque las versiones recientes (records, var) lo alivian.
- El ecosistema empresarial (Spring, Jakarta) es un mundo propio con curva considerable.
