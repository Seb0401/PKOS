# C#

## Cuándo usarla

- Backend con ASP.NET Core, apps Windows, o videojuegos con Unity (su nicho más singular).
- Contextos donde ya existe inversión en el ecosistema Microsoft.

## Cuándo NO usarla

- Fuera del ecosistema .NET/Unity, otras opciones suelen tener más comunidad para el caso concreto.

## Ventajas

- Diseño de lenguaje moderno: incorporó antes que [[java]] muchas comodidades (propiedades, LINQ, async/await).
- .NET actual es multiplataforma, rápido y open source.
- Unity lo convierte en la puerta de entrada estándar al desarrollo de juegos.

## Desventajas

- Fuera de Windows/Unity su presencia es menor que la de Java o JavaScript en el mercado local típico.
- El ecosistema histórico (Framework vs Core vs .NET 5+) confunde al llegar.
